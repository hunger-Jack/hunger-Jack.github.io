语法

区分大小写

JavaScript中的一切都是区分大小写的，变量name和变量Name是两个不同的变量。

标识符

标识符是指：变量，函数，属性名，函数参数。标识符是按照以下规则组合起来的一个或者多个字符串：

  1. 第一个字符必须是字母或者下划线(_)或者美元符号($)。
  2. 其他字符可以是字母、数字、下划线、美元符号或者汉字(不推荐)

按照惯例标识符都使用驼峰式写法，就是第一个字母小写，之后的每个有意义的单词首字母都是大写，例如：isDone、goodsLists等等。

不能把关键字、保留字、true、false、和null作为标识符。

注释

  1. 单行注释以双斜杠开头： //注释内容

语句

JavaScript中每条语句都是以分号(;)结尾，但是不是必须的，根据自己的书写习惯决定，我自己也是不习惯写分号的。写分号的有点事如果压缩合并代码，就不用担心会出错，也可以增加谢解析器的性能，应为它不用去推测应该在那一句中添加分号了。

虽然在if条件句中只有在多条语句执行的时候才使用代码块，如果只有一条可以省略花括号，但是还是推荐不管是几条语句都加上花括号，这样会使用编码看上去更清晰明了，降低胆码出错的概率。

关键字和保留字

JavaScript规定一个有特定功能的的关键字，例如控制语句的开始和结束，或者执行一些特定的操作。按照规则，这写关键字不能作为标识符。

JavaScript还规定一些不能作为标识符的保留字，这些在目前还没有什么特殊用途，但是以后可能会用到，所以叫做保留字。

数据类型

typeof操作符

  1. typeof是用来检测数据类型的，返回的是一个数据类型字符串。对一个值使用typeof操作符可能会返回以下几个字符串：

  2. “undefined”——如果这个值没有被定义

  3. “boolean”——如果这个值是布尔值
  4. “string”——如果这个值是一个字符串
  5. “number”——如果这个值是一个数值
  6. “object”——如果这个值是一个对象或者null
  7. “function”——如果这个值是一个函数

函数也是一个对象，判断一个值是否是函数的时候就可以使用typeof，但是判断一个值是否是数组最好不要使用typeof，因为数组也是对象，使用typeof返回的是”object”。除了typeof，再列举几个常用判断数据类型的方法：

  1. instanceof

    从原型链上判断。例如检测A是否为B的示例，A instanceof B,如果A是B的实例就返回true，否则返回false。[] instanceof Array 返回true，但是，由于Array是Object的一个子类，也可以说[]是一个对象，以上例子可以证明。
    这是因为[]的 __proto__ 属性指向Array的prototype，而Array.prototype.__proto__ 又指向Object.prototype,Object.prototype.__proto__指向null，标志着原型链结束。

  2. Object.prototype.toString.call(xxx)

    这个是最靠谱的一个方法了。

Undefined类型

  1. Undefined只是一个值，。在使用var声明一个变量并且没有对它初始化，那么，它的默认值就是undefined。
  2. 在使用typeof检测一个已声明未初始化的变量和一个未声明变量的类型的时候，都是会返回undefined。
  3. 如果访问一个未声明的变量，会报错，但是声明一个已声明未赋值的变量就不会报错，会返回undefined。

Null类型

  1. null是一个空指针对象。
  2. 如果声明的变量要存储对象，那么，最好把这个变量初始化为null，不仅体现null作为空指针对象的惯例，还可以很好区分null和undefined。

Boolean类型

  1. Boolean只有两个值：true和false。
  2. Boolean的两个是区分大小写的。
  3. 在执行流程控制语句的时候回自动执行Boolean类型转换。
  4. 最重要的就是Boolean类型转换规则：
数据类型 转为true的值 转为false的值
Boolean true false
String 非空字符串 空字符串
Number 任何非零数字（包括无穷大Infinity） 零和NaN
Object 任何对象 null
Undefined n/a undefined

Number类型

数值字面量格式

  1. 十进制

  2. 八进制。要求必须是0开头，然后是八进制数字序列（0-7）.如果数值中有超过数字序列范围，那么，前置0将被忽略，后面数位将按照十进制j解析。

  3. 十六进制。要求开头必须是0x开头，后跟任何十六进制数字（0-9以及A-F），字母大小写都可以。

在进行算数运算时，所有以八进制或者十六进制表示的数值最终被转换为十进制进行计算。

浮点数值

  1. 所谓浮点数值，就是该数值必须包含一个小数点，并且小数点后面至少有一位数字。虽然小数点前面可以没有数字，但是不推荐这种写法。

  2. 由于保存浮点数值需要的内存空间是保存整数的两倍。因此，如果小数点后面没有跟任何数字或者小数点后面都是0，那么改值将会被转为整数。看例子：

  3. 使用e科学计数法表示极大或者极小的数值。表示的值等于e前面的数值乘以10指数次幂，即前面是一个数值，中间是一个大写的或者小写的e，后面是10的幂中指数，看例子：

  4. 不要测试某个特定的浮点s数值，因为0.1+0.2 !== 0.3

>     关于浮点数值计算产生舍入误差问题，有一点要明确：这是使用基于IEEE754数值的浮点    计算的通病，ECMAScript并非独此一家，    所有使用相同数值格式的语言都存在这种问题。

数值范围

  1. 由于内存限制，EMACscript不能保存所有的数值。
  2. 能表示的最大的值存在Number.MIN_VALUE中，在大多数浏览器中，它的值为 5e-324;
  3. 能表示最小的值存在Number.MAX_VALUE中，在大多数浏览器中，它的值为 1.7976931348623157e+308.
  4. 只要计算出的结果超出这两个值得范围，就会返回Infinity或者-Infinity。
  5. 如果一个值返回Infinity，那么，它将不会继续参与计算，因为Infinity不能够参与数值运算。
  6. 如果要确定一个数值是否为Infinity，可以使用函数isFinite(),这个函数的参数位于+Infinity和-Infinity之间就会返回true。
  7. Number.POSITIVE_INFINITY保存Infinity，Number.NEGATIVE_INFINITY保存-Infinity

NaN

  1. NaN是一个特殊值，表示本应该返回一个数值的操作但是未返回数值的情况（防止报错），可以让代码正常运行。
  2. 任何有NaN参与的运算都会返回NaN。
  3. NaN与任何值都不相等，包括自己。

  4. isNaN()函数来判断参数是否“不是数值”，参数可以是任何类型。

数值转换

有3个函数可以把非数值转为数值：Number()、parseInt()、parseFloat()。第一个可以用于任何类型的转换，后面两种只能用于把字符串转为数值。

Number()

  1. 如果是Boolean，true转为1，false转为0。
  2. 如果是数值，按照数值字面量格式直接转为对应的十进制数值。
  3. 如果是null，转为0。
  4. 如果是undefined，转为NaN。

  5. 如果是字符串，分以下几种情况：

    1. 如果字符串全是数字（包括前面带正号或负号），则直接转换为十进制数值【注意：前导0会被忽略】。

    2. 如果字符串全是数字，并且符合浮点数格式（包括前面带正号或负号），则将其转为对应的浮点数值【注意：前导0会被忽略】。

    3. 如果字符串符合十六进制，则会转为对应的十进制数值。

    4. 如果是空字符串，则转为0。

    5. 如果字符串是除以上格式之外的情况，则都返回NaN。
  6. 如果是对象，则会先调用对象的valueOf()方法，然后按照前面的规则转换，如果转换为NaN，则再调用对象的toString()方法，然后依然按照前面规则转换，然后返回字符串。

ParseInt()

由于Number()比较负责并且有时候并不合理，所以，我们在处理整数的时候更多情况下使用的是parseInt()。parseInt()函数在转换时，更多的看字符串是否符合数值格式。

  1. parseInt()会忽略字符串前面的空格，直至找到一个非空格字符，如果第一个非空格字符不是数字字符或者负号，那么，就会返回NaN。

  2. 如果是第一个非空格字符串是数字，它会继续解析第二个，直到解析完所有字符或者遇到非数字字符。

  3. 如果字符串是以’0x’开头并且后面跟至少一个十六进制的数值，它就会被当做十六进制数值解析为十进制数值返回。

  4. 如果字符串是以’0’开头，并且后面跟至少一个八进制数值，它就会被当做八进制数值解析为十进制数值返回。

由于在EMACScript 3 JavaScript 引擎中，parseInt(‘070’)被当做八进制来解析，但是在EMACScript 5 JavaScript 引擎中，parseInt()已经不具备解析八进制值得能力，为了解决此问题，在使用parseInt()函数时，最好加一个参数，来规定按照转换时使用的基数（即多少进制），例如：

parseFloat()

parseFloat()全部按照十进制规则来解析，所有不会有第二个参数。

  1. 与parseInt()情况相同。

  2. 如果是第一个非空格字符串是数字，它会继续解析第二个，直到解析完所有字符或者遇到无效的浮点数字符。也就是说字符串的一个小数点是有效的，第二个小数点是无效的，从而它后面的字符也都不会去解析。

  3. 所有’0’开头的十六进制字符都会被转为0。

  4. 所有八进制字符串都会按照忽略前导0正常转换。

  5. 如果字符串是一个可解析为整数的数（没有小数点，或者小数点后面都是0），会返回整数。

String类型

String类型是由零个或者多个16位的Unicode字符组成的字符序列，即字符串。在ECMAScript中，字符串可以使用单引号（’’）或者双引号（””）表示。

字符字面量

String数据类型包含一些特殊的字符字面量，也叫做转义序列。用于表示非打印字符，或者具有其他用途。

字面量 含义
\n 换行
\b 空格
\t 制表
\f 换页
\r 回车
\ 反斜杠
\‘ 单引号
\” 双引号
\xnn 由十六进制代码nn表示一个字符（其中nn为0~F）。例如：\x41表示”A”
\unnnn 由十六进制代码nnn表示一个Unicode字符（其中nn为0~F）。例如：\x03a3表示希腊字母”Σ”

这些字面量可以放在字符串的任何位置，也可以被作为一个字符来解析，长度为1。

如果这个字符串里面包括双字节字符，那么length属性可能不会精确返回字符串中字符的数目。

Object类型

ECMAScript中的对象就是一种数据和功能的集合。对象可以通用new操作符后跟要创建的对象类型名称创建。而创建Object类型的实例并为其添加属性和方法，就可以创建自定义对象。例如：

Object是所有它的实例的基础，也就是说他有的属性和方法，它的实例全都有。Object有以下属性和方法：

  • Construtor：保存着创建当前对象的函数，也就是当前对象的原型。
  • hasOwnProperty(propertyName)：检查给定的属性是否存在于当前的实例(注意，不是其构造函数中），所传的属性名必须以字符串形式指定（例如：hasOwnProperty(‘name’))。
  • isPrototypeOf(obj)：用于检查给定对象是否是当前实例原型。
  • propertyIsEnumerable(propertyName)：用于检查给定的属性是否可枚举（for in）,参数同样需要以字符串形式传入。
  • toLocaleString()：返回对象的字符串表示，该字符串与执行环境的地区对应（此属性很有用，有机会再说）。
  • toString()：返回对象的字符串表示。
  • valueOf()：返回对象的字符串、数值、布尔值表示，通常与toString()方法返回的值相同。

从技术角度来讲，ECMA-262中对象的行为不一定适用JavaScript中的其他对象。浏览器环境中的对象，比如DOM和BOM中的对象，他们属于宿主对象，因为他们是由宿主实现提供和定义的。ECMA不负责定义宿主对象，所以宿主对象可能会也可能不会继续Object。
